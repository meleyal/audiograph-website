Introduction

Welcome to the AudioGraph API documentation. Our API provides comprehensive audio processing capabilities through simple REST endpoints. 

curl -X POST \
  'https://api.audiograph.org/v1/transcribe' \
  -H 'Authorization: Bearer YOUR_API_KEY' \
  -H 'Content-Type: multipart/form-data' \
  -F 'file=@audio.mp3' \
  -F 'webhook_url=https://example.com/webhook'

Authentication

All API requests require authentication using a Bearer token in the Authorization header. 

Authorization: Bearer YOUR_API_KEY

Errors

Code Description
400 Bad Request - Invalid parameters
401 Unauthorized - Invalid API key
429 Too Many Requests - Rate limit exceeded

Transcoding

Convert audio files between different formats with optimal settings. 

curl -X POST \
  'https://api.audiograph.org/v1/transcode' \
  -H 'Authorization: Bearer YOUR_API_KEY' \
  -H 'Content-Type: multipart/form-data' \
  -F 'file=@audio.wav' \
  -F 'format=mp3' \
  -F 'quality=320' \
  -F 'webhook_url=https://example.com/webhook'

Tagging

Automatically identify and tag audio content with high accuracy. Tags include genres, moods, instruments, and other audio characteristics. 

curl -X POST \
  'https://api.audiograph.org/v1/tag' \
  -H 'Authorization: Bearer YOUR_API_KEY' \
  -H 'Content-Type: multipart/form-data' \
  -F 'file=@audio.mp3' \
  -F 'include_confidence=true' \
  -F 'webhook_url=https://example.com/webhook'

Source Separation

Split audio into individual stems including vocals, drums, bass, and other instruments. Perfect for remixing and audio analysis. 

curl -X POST \
  'https://api.audiograph.org/v1/separate' \
  -H 'Authorization: Bearer YOUR_API_KEY' \
  -H 'Content-Type: multipart/form-data' \
  -F 'file=@song.mp3' \
  -F 'stems[]=vocals' \
  -F 'stems[]=drums' \
  -F 'stems[]=bass' \
  -F 'stems[]=other' \
  -F 'webhook_url=https://example.com/webhook'

Transcription

Convert speech to text with support for multiple languages, speaker diarization, and punctuation. Ideal for podcasts, interviews, and voice notes. 

curl -X POST \
  'https://api.audiograph.org/v1/transcribe' \
  -H 'Authorization: Bearer YOUR_API_KEY' \
  -H 'Content-Type: multipart/form-data' \
  -F 'file=@interview.mp3' \
  -F 'language=en' \
  -F 'diarization=true' \
  -F 'webhook_url=https://example.com/webhook'

Genre Classification

Identify musical genres and subgenres with high accuracy. Our model recognizes over 500 different genres and styles. 

curl -X POST \
  'https://api.audiograph.org/v1/classify/genre' \
  -H 'Authorization: Bearer YOUR_API_KEY' \
  -H 'Content-Type: multipart/form-data' \
  -F 'file=@track.mp3' \
  -F 'include_subgenres=true' \
  -F 'webhook_url=https://example.com/webhook'

BPM Detection

Accurate tempo detection and beat tracking for any musical content. Returns BPM, beat positions, and rhythm confidence scores. 

curl -X POST \
  'https://api.audiograph.org/v1/detect/bpm' \
  -H 'Authorization: Bearer YOUR_API_KEY' \
  -H 'Content-Type: multipart/form-data' \
  -F 'file=@track.mp3' \
  -F 'return_beats=true' \
  -F 'webhook_url=https://example.com/webhook'

Webhooks

Get notified when your audio processing is complete through webhooks. We'll send a POST request to your specified URL with the results. 

{
  "id": "job_123",
  "status": "completed",
  "result": {
    "bpm": 128,
    "confidence": 0.95,
    "beat_positions": [0.5, 1.0, 1.5]
  }
}

Polling

If you prefer not to use webhooks, you can poll the job status endpoint to check when processing is complete. Each job returns a unique ID that you can use to check its status. 

curl -X GET \
  'https://api.audiograph.org/v1/jobs/job_123' \
  -H 'Authorization: Bearer YOUR_API_KEY'

The response will include the job's current status and results if available: 

{
  "id": "job_123",
  "status": "completed", // or "processing", "failed"
  "created_at": "2025-01-01T12:00:00Z",
  "completed_at": "2025-01-01T12:01:30Z",
  "result": {
    // endpoint-specific results here
  },
  "error": null
}

Assets

Many endpoints return signed URLs to processed assets in their response. These assets are stored securely and the URLs expire after 30 days. 

{
  "id": "job_123",
  "status": "completed",
  "result": {
    "stems": {
      "vocals": "https://assets.audiograph.org/stems/vocals_j4k2l3.mp3?signature=abc123...",
      "drums": "https://assets.audiograph.org/stems/drums_h8k2m4.mp3?signature=def456...",
      "bass": "https://assets.audiograph.org/stems/bass_n9p4q7.mp3?signature=ghi789..."
    }
  }
}

Testing

Use your test API key to safely test your integration. Test keys return example responses without performing any actual processing, and you won't be billed for these requests. 

curl -X POST \
  'https://api.audiograph.org/v1/transcribe' \
  -H 'Authorization: Bearer test_YOUR_API_KEY' \
  -H 'Content-Type: multipart/form-data' \
  -F 'file=@audio.mp3'

Test API keys start with test_. You'll receive standardized example responses that match the structure of production responses: 

{
  "id": "job_test_123",
  "status": "completed",
  "result": {
    "text": "This is an example transcription response.",
    "confidence": 0.95,
    "language": "en",
    "speakers": [
      {"id": 1, "start": 0.0, "end": 2.5},
      {"id": 2, "start": 2.8, "end": 5.0}
    ]
  }
}

Batch Processing

Batch processing allows you to process multiple audio files at once. This is useful for large-scale projects or when you need to process multiple files simultaneously. 

curl -X POST \
  'https://api.audiograph.org/v1/transcribe' \
  -H 'Authorization: Bearer YOUR_API_KEY' \
  -H 'Content-Type: multipart/form-data' \
  -F 'file=@audio1.mp3' \
  -F 'file=@audio2.mp3' \
  -F 'file=@audio3.mp3' \
  -F 'webhook_url=https://example.com/webhook'

Real-time Processing (Beta)

Process audio streams in real-time using our WebSocket API or WebRTC. Perfect for live events, streaming applications, and interactive experiences. WebRTC offers lower latency and better quality for browser-based applications.

WebSocket

const socket = new WebSocket(
  'wss://api.audiograph.org/v1/realtime',
  ['Bearer', 'YOUR_API_KEY']
);

const response = await fetch('https://api.audiograph.org/v1/realtime', {
    method: 'POST',

      'Content-Type': 'application/json'
    },
    body: offer.sdp,
